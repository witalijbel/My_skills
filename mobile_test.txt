Рекомендации к приложениям магазина APP STORE Guidlines

1. Родительский контроль над приложениям
 1.1 Категория "Дети"
  1.1.1. Приложение должно соответствовать требованиям детской категории 
  1.1.2. Приложение должно продолжать соответствовать требованиям при обновлениях
 1.2. Приложения в категории «Дети» не должны включать стороннюю аналитику или стороннюю рекламу
 1.3. Приложения не должны содержать ссылки из приложения, возможности покупок или другие отвлекающие факторы для детей
2. Рассмотрение возможность использования Xcode для бесплатной установки приложения
3. Уважение пользователей с различными мнениями
4. НЕ пытаться обмануть систему
 4.1. Обмануть процесс проверки
 4.2. Украсть пользовательские данные
 4.3. Скопировать работу другого разработчика
 4.4. Манипулировать рейтингами
5. Приложения не должны включать в себя контент, который является оскорбительным
 5.1. Содержать методы фильтрации нежелательных материалов, не публикуемых в приложении
 5.2. Механизм для сообщения оскорбительного содержания и своевременных ответов на вопросы
 5.3. Возможность заблокировать оскорбительных пользователей от сервиса
6. Безопастность данных
 6.1. Реализация соответствующих мер безопасности для обеспечения надлежащего обращения с пользовательской информацией
7. Приложение должно включать простой способ обратной связи с разработчиком
8. Поведение приложения не должно быть сопряжено с риском нанесения физического вреда пользователю
9. Совместимость оборудования
 9.1. Приложения для iPhone по возможности должны запускаться на iPad
 9.2. Приложения никогда не должны предлагать или требовать перезапуск
 9.3. Приложения не должны быстро разряжать аккумулятор, чрезмерно нагреваться или чрезмерно загружать ресурсы устройства
 
 Рекомендации к приложениям магазина Android Guidlines
 
 1. Визуальный дизайн
  1.1. Приложение должно обеспечивать стандартный визуальный дизайн Android
  1.2. Приложение должно иметь шаблоны взаимодействия там, где это необходимо, для единообразного и интуитивно понятного взаимодействия с пользователем.
 2. Приложение должно реализовывать ожидаемое функциональное поведение 
 3. Приложение должно обеспечивать производительность, стабильность, совместимость и отзывчивость, ожидаемые пользователями
 4. Приложение должно безопасно обрабатывать пользовательские данные и личную информацию с соответствующим уровнем разрешений
  4.1. Приложение должно соответствовать политике данных пользователя для защиты конфиденциальности пользователей
 5. Приложение должно соответствовать основным требованиям к качеству. В частности, для поддержки устройств с большим экраном в альбомной и книжной ориентации
 6. Приложение должно минимизировать свою активность в фоновом режиме и когда устройство работает от батареи
 7. Сделайте приложение доступным для работы в автономном режиме
